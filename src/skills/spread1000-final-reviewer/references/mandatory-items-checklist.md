# 必須記載事項チェックリスト

研究計画調書（様式1）の必須記載事項を検証するための詳細チェック基準。

## I. 研究目的・研究方法

### チェックポイント
- [ ] 研究の目的が明確に記述されている
- [ ] 対象とする現象・課題が具体的に特定されている
- [ ] 解決しようとする問題が明確に定義されている
- [ ] 得ようとする知見が具体的に示されている
- [ ] AI適用の各工程（データ取得→前処理→学習→推論→評価）が段階的に記載されている
- [ ] 使用するデータの**種類**が明記されている
- [ ] 使用するデータの**出所**（公開データセット、独自収集等）が明記されている
- [ ] データの**取得方法**が明記されている
- [ ] 評価指標（精度、再現率、F1等）が定義されている
- [ ] 検証方法（交差検証、ホールドアウト等）が記載されている

### 不合格条件
- 研究目的が抽象的で「何を明らかにするか」が不明
- AIをどの工程でどう使うかの記載がない
- データの出所が不明

## II. AI利活用の妥当性・実現可能性

### チェックポイント
- [ ] 従来手法（非AI）の限界が具体的に記載されている
- [ ] AIを導入することで解決可能になる事項が明示されている
- [ ] AI利活用の**必要性**（なぜAIでなければならないか）が論理的に説明されている
- [ ] AI利活用の**妥当性**（提案手法が適切である根拠）が示されている
- [ ] 技術的実現可能性の根拠（先行研究、予備実験等）がある
- [ ] 単なる既存手法の置き換えではなく、AI ならではの革新性が示されている

### 不合格条件
- 「AIを使うと効率的」程度の記載で、具体性がない
- 従来手法との比較がない
- 実現可能性の根拠がない

## III. 達成目標

### チェックポイント
- [ ] **中間目標（3ヶ月後）** が明確に記載されている
- [ ] **最終目標（6ヶ月後）** が明確に記載されている
- [ ] 中間・最終の区分が明示的に分かれている
- [ ] 目標が具体的で測定可能（定量的指標がある）
- [ ] PoC・予備検証等を含め、研究の前進が示されている
- [ ] 目標と研究方法の整合性がある

### 不合格条件
- 中間・最終の区分がない
- 目標が抽象的で達成度の判断ができない

## IV. ノウハウ抽出・共有計画

### チェックポイント
- [ ] AI利活用の知見を**どのように整理・抽出するか**が記載されている
- [ ] **共有方法**（学会発表、論文、ワークショップ、GitHub等）が具体的
- [ ] **他分野への展開可能性**が言及されている
- [ ] SPReADコミュニティでの共有が意識されている
- [ ] 各工程の課題と解決策を体系的に整理する計画がある

### 不合格条件
- 「論文を書く」程度の記載で具体性がない
- 他分野展開の視点がない

## V. 研究業績

### チェックポイント
- [ ] 学術論文・学会発表・著書から **合計5件以内** で記載
- [ ] 各項目が箇条書きになっている
- [ ] 著者名に本人が含まれ、**下線**が付されている
- [ ] 学術論文: 題名、著者、雑誌名、発行年月が記載
- [ ] 学会発表: 題名、著者、学会名、発行年月が記載
- [ ] 当該研究課題との**関連性**が読み取れる業績が選ばれている
- [ ] 被引用数を記載する場合は情報源と取得日が明記されている

### 不合格条件
- 6件以上記載されている
- 著者に下線がない
- 当該研究課題と関連のない業績のみ

## VI. AIインタビュー完了確認

### チェックポイント
- [ ] 完了メールのスクリーンショットが5枚目に貼付されている
- [ ] スクリーンショット内にメールアドレスが表示されている
- [ ] メールアドレスが研究計画調書の登録アドレスと一致
- [ ] 所属機関発行のメールアドレスで登録されている

### 不合格条件（応募不受理）
- スクリーンショットがない
- メールアドレスが確認できない
- メールアドレスが不一致

## VII. 計算資源確保計画

### チェックポイント
- [ ] **調達方法**が記載されている（Azure, AWS, オンプレミス, HPCI等）
- [ ] **利用開始時期**が記載されている
- [ ] **利用期間**が記載されている
- [ ] **想定規模**（GPU数、ストレージ容量等）が記載されている
- [ ] HPCI「特定研究有償課題」の活用が検討されている
- [ ] Azure構成設計書（Phase 1）との整合性がある

### 不合格条件
- 計算資源の確保方法が全く記載されていない

## VIII. 他の研究費の応募・受入状況

### チェックポイント
- [ ] e-Radでの申告が必要であることが認識されている
- [ ] 同一経費の重複計上がないことが確認されている
- [ ] ARiSE等との重複制限を確認済み

### 不合格条件
- 重複計上の疑い

## IX. 研究インテグリティ

### チェックポイント
- [ ] e-Rad上で誓約状況が登録済み（未登録は応募不受理）
- [ ] 研究データ管理の方針が示されている

### 不合格条件（応募不受理）
- e-Radで誓約未登録

## X. 研究計画の変更対応

### チェックポイント
- [ ] 早期達成時の深化・高度化の方針が記載されている
- [ ] 想定外の困難が発生した場合の代替プランが示されている

### 不合格条件
- 早期達成・計画変更への言及がない

# Post-Award Manager

SPReAD 採択後の交付申請から研究成果報告書の提出までを一貫してガイドする。

## Use This Skill When

- 採択通知を受け取った後の手続きを知りたい
- 交付申請手続きの進め方を確認したい
- 中間進捗報告（3ヶ月時点）を作成したい
- 最終成果報告書（6ヶ月時点）を作成したい
- 会計実績報告書を作成したい
- 予算の費目間流用や経費配分変更を行いたい
- 研究計画を変更したい
- 論文に謝辞（Acknowledgment）を書きたい
- 研究データマネジメントプランの実施状況を確認したい
- 早期に目標達成した場合の対応を知りたい

## Required Inputs

- 採択課題名・e-Rad 課題番号
- 交付決定日（研究期間の起点）
- 研究代表者情報（氏名・所属機関）
- 研究計画書の最新版（`output/phase3-proposal.md` があれば参照）

## Workflow

### Step 1: 採択後ロードマップ生成

交付決定日を起点に180日間の管理スケジュールを生成し `output/post-award-roadmap.md` へ保存。

```
■ 採択後ロードマップ（180日間）

交付決定日      : YYYY-MM-DD
研究期間終了日  : 令和9年1月6日

【交付申請フェーズ】（交付決定後 速やかに）
  □ 交付申請書の提出（機関等経由）
  □ 研究倫理教育・コンプライアンス教育の受講確認
  □ 研究データマネジメントプラン及びチェックリストの提出
  □ 機関等向け AI利活用に係る研究データ取扱いチェックリストの提出

【中間マイルストーン】（3ヶ月時点: YYYY-MM-DD 頃）
  □ 中間目標の達成状況を自己点検
  □ 中間進捗メモの作成
  □ 予算執行状況の確認（費目間流用の必要性判断）
  □ 早期達成時: 研究の深化・高度化方針を検討

【最終報告フェーズ】（6ヶ月時点: 令和9年1月6日）
  □ 最終目標の達成状況を整理
  □ 研究成果報告書の作成
  □ 会計実績報告書の作成
  □ 研究データマネジメントプランの最終更新
  □ メタデータの付与・登録（NII Research Data Cloud 等）

【提出期限】
  □ 会計実績報告書・研究成果報告書: 令和9年2月上旬（予定）
  □ 間接経費使用実績: 令和9年6月30日まで（機関等がe-Radで報告）
```

### Step 2: 交付申請手続きガイド

採択通知後に必要な手続きを案内。以下の必須タスクを `output/grant-application-checklist.md` へ保存:

| # | タスク | 実施者 | 提出先 | 備考 |
|---|--------|--------|--------|------|
| 1 | 交付申請書の作成・提出 | 機関等 | 文部科学省 | 採択通知に記載の期日まで |
| 2 | 研究倫理教育の受講証明 | 研究代表者 | 機関等 | 交付申請時までに完了必須 |
| 3 | コンプライアンス教育の受講証明 | 研究代表者 | 機関等 | 交付申請時までに完了必須 |
| 4 | 研究データマネジメントプラン及びチェックリスト | 研究代表者 | 文部科学省（機関等経由） | FAIR原則に基づく |
| 5 | AI利活用に係る研究データ取扱いチェックリスト | 機関等 | 文部科学省 | 機関等向け様式 |
| 6 | researchmap 登録情報の更新 | 研究代表者 | researchmap | 積極的登録を推奨 |

### Step 5: 予算変更・費目間流用ガイド

予算執行中に費目間の流用や経費配分変更が必要になった場合の手続きを案内し、`output/budget-change-guide.md` へ保存:

```markdown
# 予算変更・費目間流用ガイド

## 費目間流用（研究代表者判断で可能な範囲）

直接経費の各費目間での配分変更のうち、交付要綱に定める「費目間流用の範囲」内
であれば、研究代表者の判断で調整可能。

### 手続きフロー
1. 現在の予算配分と執行状況を確認
2. 流用額が「費目間流用の範囲内」か判定
3. 範囲内 → 研究代表者の責任で実施（事後的に会計実績報告で報告）
4. 範囲外 → 経費配分変更承認申請書を機関等経由で文部科学省に提出

## 経費配分変更承認申請（文部科学省の承認が必要）

以下のいずれかに該当する場合は、事前に承認申請が必要:
- 費目間流用の範囲を超える変更
- 研究計画の本質的な変更を伴う経費配分の見直し

### 必要書類
- 経費配分変更承認申請書
- 変更理由書（変更の必要性と研究への影響を記載）

### 提出先
機関等を通じて文部科学省へ提出

## 注意事項
- 間接経費（直接経費の30%）は変更不可
- 対象外経費への流用は不可（建物等施設、土地購入費、不動産取得税等）
- 変更後も補助上限500万円（直接経費）を超えないこと
```

### Step 6: 研究計画変更手続きガイド

研究の進展に伴い計画変更が必要になった場合の手続きを案内:

```
■ 研究計画変更のケースと対応

【ケース A: 早期目標達成】
→ 研究の深化・高度化に取り組む
→ 変更内容を記録し、最終報告書に反映

【ケース B: 研究方法・アプローチの変更】
→ 軽微な変更: 研究代表者の判断で実施、報告書に記載
→ 本質的変更: 研究計画変更承認申請書を機関等経由で文部科学省へ提出

【ケース C: 研究中止・辞退】
→ 速やかに機関等に連絡し、文部科学省へ届出
→ 既交付分の精算・返還手続き

必要書類: 研究計画変更承認申請書（変更理由・変更後の計画を記載）
```

## Quality Gates

- [ ] 交付申請に必要な全タスクがチェックリスト化されている
- [ ] 中間進捗メモに予算執行率が含まれている
- [ ] 最終報告書にAI活用の具体的成果が記載されている
- [ ] 論文謝辞の体系的番号（JPMXP17 + e-Rad課題番号）が正しい
- [ ] 会計実績報告書の費目合計が交付決定額以内である
- [ ] 研究データマネジメントの FAIR原則適合状況が記載されている

## Gotchas

- 研究成果報告書を提出しないと補助金の交付決定取消・返還命令の対象になる
- 費目間流用の「範囲内」は交付要綱で定められる。公募要領には具体的割合が記載されていないため、交付決定時の通知を確認すること
- 間接経費（30%）は研究代表者ではなく機関等が管理・報告する
- 令和7年度補正予算での実施のため、通常の科研費等とは手続きスケジュールが異なる場合がある
- 論文謝辞の体系的番号を誤ると研究成果が本事業と紐付かない
- 研究データの公開にあたっては、オープン・アンド・クローズ戦略を踏まえ、非公開とする正当な理由がある場合は非公開も可
- 本事業では伴走支援（情報共有・コミュニティ形成・好事例の横展開等）が実施される。機関等のURA等と連携し円滑に参画すること

## Validation Loop

1. 各テンプレート（ロードマップ・報告書・会計）を生成する
2. Check:
   - 研究期間（180日間）とマイルストーン日付が整合しているか
   - 予算額が交付決定額以内か
   - 体系的番号（JPMXP17 + e-Rad課題番号 = 15桁）が正しいか
   - 研究データマネジメントのFAIR原則適合が記載されているか
3. If any check fails:
   - 日付・金額・番号を修正
   - 再検証する
4. 全ゲートをパスした後のみ成果物を最終化する

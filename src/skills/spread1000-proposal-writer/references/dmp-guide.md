# 研究データマネジメント計画（DMP）ガイド

SPReAD 申請書の研究データ管理方針記載に関するガイドライン。

## 研究計画調書での記載事項

公募要領では、使用するデータについて以下を明確に記載することが求められている:
- データの **種類**
- データの **出所**
- データの **取得方法**
- 利用に当たっての **留意事項**

## DMP記載テンプレート

### 1. 研究データの概要

| 項目 | 内容 |
|------|------|
| データの種類 | <!-- 例: 分子シミュレーションデータ、画像データ、テキストコーパス --> |
| データの形式 | <!-- 例: CSV, HDF5, DICOM, FITS --> |
| データの規模 | <!-- 例: 約100GB、100万レコード --> |
| データの出所 | <!-- 例: 自前生成、公開データベース（PDB, ChEMBL等）、共同研究者提供 --> |
| データの取得方法 | <!-- 例: 実験測定、シミュレーション生成、API取得、購入 --> |

### 2. データ管理方針

| 項目 | 方針 |
|------|------|
| 保存場所 | <!-- 例: Azure Blob Storage（暗号化・アクセス制御付き） --> |

| データ公開 | <!-- 例: 論文投稿時にリポジトリで公開、匿名化処理後 --> |

### 3. 倫理的配慮

| 項目 | 対応 |
|------|------|
| 個人情報の有無 | あり / なし |

| インフォームドコンセント | 取得済み / 不要 |
| データの匿名化 | 実施 / 不要 |
| 法令遵守 | <!-- 例: 個人情報保護法、生命倫理指針 --> |
